# **Посібник для AGENTS**

Цей документ описує мінімальну реалізацію агента OpenAI (assistant), яка входить до репозиторію **openai‑assistant**. Проєкт створено для навчальних цілей: простота коду та зрозумілість його структури важливіші за повну функціональність. Файл має допомогти вам швидко зрозуміти, які компоненти є у проєкті, як вони взаємодіють між собою і як налаштувати та запустити агента.

## **Огляд проєкту**

Проєкт реалізує консольного асистента на основі API OpenAI. Асистент може:

1. **Завантажувати файли** до OpenAI, щоб використовувати їх як джерело знань.

2. **Створювати та підключати векторне сховище** (vector store) для пошуку інформації у завантажених файлах.

3. **Створювати або знаходити асистента** за іменем і передавати йому векторне сховище.

4. **Створювати або знаходити тред (thread)** для збереження історії діалогу.

5. **Додавати повідомлення користувача до треду** й за потреби прикріплювати файл.

6. **Запускати асистента** на обраному треді та отримувати відповідь.

За основу взято API «Assistants» OpenAI; він буде закритий 26 серпня 2026 року, однак підходи, що використовуються в коді, можуть залишатися корисними для розуміння роботи з векторними сховищами та потоками [platform.openai.com.](http://platform.openai.com) Векторне сховище дає можливість шукати дані у файлах: OpenAI самостійно розбиває текст на фрагменти, створює вектори та виконує пошук за ключовими словами й смисловою близькістю 

## **Структура директорій**
``` bash
openai‑assistant/
├─ .env.examle           – шаблон файлу налаштувань (API‑ключ, назви файлів тощо)
├─ files/                – папка, куди додаються власні файли (наприклад, ваш розклад)
├─ src/
│  ├─ index.js           – точка входу програми
│  ├─ assistant_manager.js – логіка створення/пошуку асистента та оновлення його інструкцій
│  ├─ file_manager.js    – логіка завантаження файлів та створення/очищення векторного сховища 
│  ├─ thread_manager.js  – логіка створення тредів, додавання повідомлень і читання останньої відповіді
│  ├─ run_manager.js     – запуск асистента на треді й відстеження статусу run
│  └─ utils.js           – допоміжні функції (форматування дат)
└─ assistant_instructions.md – файл із інструкціями для асистента (створюється вами)
```

### **index.js**

У `index.js` описано послідовність кроків, що виконуються під час запуску. Файл імпортує модулі з папки `src` та виконує такі дії:

1. **Читає змінні оточення** з `.env` (API‑ключ, назви файлів, моделі тощо).

2. **Отримує або створює файл** за допомогою `getFileId()`. Якщо файл ще не завантажено до OpenAI, він завантажується.

3. **Отримує або створює асистента** (`getAssistantId()`). Якщо асистент з певним іменем не існує, створюється новий з інструкціями з `assistant_instructions.md` та інструментом `file_search`.

4. **Створює векторне сховище** для завантажених файлів (`getVectorStoreId()`) і підключає його до асистента (`updateAssistantWithVectorStore()`). Згідно з документацією, один асистент може мати лише одне векторне сховище[learn.microsoft.com](https://learn.microsoft.com/en-us/azure/ai-foundry/openai/how-to/file-search#:~:text=Vector%20stores), тому перед створенням нового сховища попередні сховища з тим самим ім’ям видаляються.

5. **Створює новий тред** (`getThreadId()`), який зберігає історію діалогу.

6. У нескінченному циклі зчитує запит користувача з консолі, додає його до треду (`addMessageToThread()`), запускає асистента (`runAssistantOnThread()`), чекає завершення run та виводить відповідь (`getLastResponse()`).

### **assistant\_manager.js**

Містить функції для роботи з асистентом:

* `getAssistantId(openAiInstance, assistantName)` – шукає асистента за іменем. Якщо кілька асистентів мають однакове ім’я, обирається найновіший. Якщо жодного немає, створюється новий асистент із заданою моделлю, температурою та інструкціями з файлу `assistant_instructions.md`.

* `updateAssistantWithVectorStore(openAiInstance, assistantId, vectorStoreId)` – підключає векторне сховище до асистента через параметр `tool_resources`. Саме цей параметр дозволяє «прикріпити» сховище до асистента або треду[learn.microsoft.com](https://learn.microsoft.com/en-us/azure/ai-foundry/openai/how-to/file-search#:~:text=Attaching%20vector%20stores).

### **file\_manager.js**

Відповідає за роботу з файлами та векторними сховищами:

* `getFileId(openAiInstance, fileName)` – шукає файл за назвою. Якщо його немає, читає файл із папки `files/` та завантажує до OpenAI. Для нового файлу використовується призначення `assistants`.

* `getVectorStoreId(openAiInstance, fileIds)` – створює нове векторне сховище з файлів (або видаляє існуюче сховище з тією ж назвою). За замовчуванням встановлюється політика автоматичного видалення через 7 днів, щоб уникнути зайвих витрат[learn.microsoft.com](https://learn.microsoft.com/en-us/azure/ai-foundry/openai/how-to/file-search#:~:text=The%20,files%20and%20their%20corresponding%20embeddings).

* Допоміжна функція `checkAndRemoveExistingVectorStore()` перевіряє наявність сховищ з однаковою назвою й очищує їх. Це важливо, оскільки API дозволяє лише одне векторне сховище для асистента[learn.microsoft.com](https://learn.microsoft.com/en-us/azure/ai-foundry/openai/how-to/file-search#:~:text=Vector%20stores).

### **thread\_manager.js**

Модуль для створення тредів і додавання повідомлень:

* `getThreadId(openAiInstance)` – створює новий тред. У цій реалізації треди не зберігаються між запуском програми.

* `addMessageToThread(openAiInstance, threadId, message, fileId?)` – додає повідомлення користувача до треду. За необхідності можна додати файл до повідомлення (рядок закоментовано у `index.js`).

* `getLastResponse(openAiInstance, threadId)` – повертає останнє повідомлення від асистента.

### **run\_manager.js**

Містить функції для керування запуском асистента:

* `runAssistantOnThread(openAiInstance, threadId, assistantId)` – запускає асистента на вказаному треді.

* `getRunStatus(openAiInstance, threadId, runId)` – перевіряє статус запуску. У разі помилки виводить опис і завершує цикл очікування.

### **utils.js**

Невеликий модуль для форматування дат (`formatDate()`). Корисний для виводу створених об’єктів у зручному вигляді.

## **Налаштування та запуск**

**Клонування репозиторію** й встановлення залежностей:

 `git clone https://github.com/24fmifyepetrushenko-source/openai-assistant.git`  
`cd openai‑assistant`  
`npm install`

1.   
2. **Створення файлів та налаштування `.env`:**

   * Скопіюйте `.env.examle` до `.env` і заповніть значення:

     * `OPENAI_API_KEY` – ваш API‑ключ OpenAI.

     * `FILE_NAME` – назва файлу у папці `files/` (наприклад, `rozklad.docx`).

     * `FOLDER_NAME` – папка з файлами (за замовчуванням `files`).

     * `ASSISTANT_NAME` – ім’я асистента (наприклад, “Schedule Assistant”).

     * `OPENAI_MODEL` – назва моделі (наприклад, `gpt‑4o-mini-2024-07-18`).

     * `OPENAI_TEMPERATURE` – температура генерації (0–2).

     * `VECTOR_STORE_NAME` – необов’язкове ім’я векторного сховища; якщо не вказане, буде створене автоматично зі штампом дати.

3. **Додайте файл інструкцій** `assistant_instructions.md` у папку `files` (або іншу папку, зазначену в `FOLDER_NAME`). У цьому файлі опишіть роль асистента, очікуваний формат відповіді тощо. Наявність інструкцій забезпечує стабільніші та контрольовані відповіді.

4. **Запуск програми:** виконайте `npm start`. У консолі з’являться повідомлення про створення/пошук файлів, асистента та треда. Далі можна вводити запити; асистент відповідатиме, використовуючи дані з файлу.
5. **Тестування програми**: Необхідно вирішити чи буде тестування з MOCK данними, або робити реальний запит до OpenAI API. Для тестування використовують наступні змінні середовища (USE_MOCK_OPENAI, TEST_USER_MESSAGE) Виконайте `npm run tet` для тестування.

### **Приклади змісту `assistant_instructions.md`**

У файлі інструкцій опишіть роль та стиль спілкування асистента. Наприклад:

`Ви – помічник розкладу. Отримавши запит від студента, надавайте інформацію про предмети та час занять на конкретний день. Відповідайте українською мовою й використовуйте лише дані, що містяться у завантаженому документі.`

## **Рекомендації та найкращі практики**

1. **Простота та модульність.** Код розділено на невеликі модулі за принципом KISS (“keep it simple, stupid”). Кожна функція виконує одну задачу й має перевірку параметрів. Такий підхід спрощує тестування та розширення.

2. **Асинхронність та обробка помилок.** Всі виклики до API обгорнуті в `try/catch` і повертають змістовні повідомлення у разі помилки. Якщо run має статус `failed`, програма виведе опис помилки і не заблокується.

3. **Готовність векторного сховища.** Після створення сховища важливо переконатись, що всі файли повністю оброблені, перш ніж запускати run. Документація радить використовувати хелпери SDK або перевіряти поле `file_counts` для впевненості[learn.microsoft.com](https://learn.microsoft.com/en-us/azure/ai-foundry/openai/how-to/file-search#:~:text=Adding%20files%20to%20vector%20stores,of%20the%20file%20ingestion%20operation).

4. **Обмеження векторних сховищ.** Можна додати лише одне сховище до асистента і одне до треду[learn.microsoft.com](https://learn.microsoft.com/en-us/azure/ai-foundry/openai/how-to/file-search#:~:text=Vector%20stores). Якщо вам потрібна нова версія сховища, видаліть попереднє і створіть нове з актуальними файлами.

5. **Керування витратами.** Сховища мають політику автоматичного видалення, яку можна змінювати. Видаляйте непотрібні файли, щоб уникати зайвих витрат на зберігання[learn.microsoft.com](https://learn.microsoft.com/en-us/azure/ai-foundry/openai/how-to/file-search#:~:text=The%20,files%20and%20their%20corresponding%20embeddings).

6. **Дотримання термінів деактивації API.** API Assistants буде вимкнуто наприкінці серпня 2026 року; обираючи цю реалізацію, зважайте на майбутнє оновлення програмного інтерфейсу.

## **Як розширити чи змінити агента**

* **Додавання нових функцій.** Якщо потрібно додати підтримку функцій (function calling) або інтеграцію з API, створіть додатковий модуль (наприклад, `function_manager.js`) та підключіть його у `index.js`. Дотримуйтеся принципу «один модуль – одна відповідальність».

* **Збереження історії тредів.** У цій реалізації кожен запуск створює новий тред. Щоб використовувати один тред для кількох сесій, зберігайте `threadId` у базі даних або файлі й передавайте його у `getThreadId()`.

* **Зміна формату виводу.** Ви можете модифікувати код у `thread_manager.js` для форматування відповіді (наприклад, кольоровий вивід, логування в файл тощо).

* **Підтримка кількох файлів.** Функція `getVectorStoreId()` приймає масив ідентифікаторів файлів, тому ви можете завантажити кілька документів і використовувати їх одночасно. Не забудьте, що кожний файл повинен бути у папці, зазначеній у `FOLDER_NAME`.

## **Висновок**

Проєкт **openai‑assistant** демонструє базову інтеграцію з API OpenAI для створення агента, який може використовувати ваші документи як джерело знань. Завдяки модульній структурі та простому коду його легко адаптувати під власні задачі. Водночас зверніть увагу на обмеження API та готуйтеся до міграції на нові версії, коли вони з’являться.
